if Statements and Control Flow
Motivation: The Need for If-Else Statements
In the world of algorithmic programming, the ability to make decisions is crucial.
Without the capability to make choices and perform different actions based on
specific conditions, algorithms would be rigid and limited. This is where if-else
statements come into play.
• They allow algorithms to adapt and respond to different situations.
• If-else statements help algorithms make decisions, just like humans do in
everyday life.
• They enable algorithms to solve problems more intelligently and efficiently.
• Learning how to use if-else statements is a fundamental skill for any
programmer or problem solver.
2/24
Syntax of If-Else Statements
Syntax:
If (<condition>) Then
... // Code to execute when the condition is true
Else
... // Code to execute when the condition is false
End If;
!
Remark:
Conditions are expressions that evaluate to true or false. Based on the condi-
tion’s result, the algorithm follows different execution paths.
3/24
Example: If-Else Statement
Let’s consider a simple algorithm to determine if a person is eligible to vote based
on their age:
Example
Algorithm Check_Voting_Eligibility;
Var age: integer;
Begin
Write("Enter your age: ");
Read(age);
If (age >= 18) Then
Write("You are eligible to vote.");
Else
Write("You are not eligible to vote.");
End If;
End;
4/24
Relational Operations
Common Relational Operations and Symbols
• Greater Than (>): It’s true if the left value is greater than the right
value.
• Less Than (<): It’s true if the left value is less than the right value.
• Greater Than or Equal To (>=): It’s true if the left value is greater
than or equal to the right value.
• Less Than or Equal To (<=): It’s true if the left value is less than or
equal to the right value.
• Equal To (=): It’s true if the left value is equal to the right value.
• Not Equal To (! =): It’s true if the left value is not equal to the right
value.
5/24
Logical Operators
Logical operators are used to combine and manipulate conditions in if-else
statements. Here are the common logical operators:
• And (&&) - Returns true if both conditions are true.
• Or (||) - Returns true if at least one condition is true.
• Not (!) - Reverses the condition; true becomes false and vice versa.
Example: Logical Operators
• If (age >= 18 && citizenship == "DZ") Then - Checks if a person
is both 18 or older and a Algerian citizen.
• If (!hasPermission) Then - Checks if a permission flag is false (i.e.,
permission is denied).

Nested If Statements
In some situations, a single ‘if‘ statement may not be sufficient to express complex
conditions. That’s where nested ‘if‘ statements come into play.
With nested ‘if‘ statements, you can include one ‘if‘ statement within another,
creating a hierarchy of conditions. This allows you to check multiple conditions,
each with its own set of actions.

Introducing the Switch Statement
So far, we’ve explored if-else statements, which are great for making decisions based
on conditions. However, in some cases, you might need to make decisions based on
the value of a single variable, choosing from a list of predefined options.
That’s where the switch statement comes into play. It provides a way to select
one option among many based on the value of an expression.
10/24
Syntax of the Switch Statement
The switch statement provides a more efficient way to handle multiple cases, and
we’ll explore its usage with examples.
Syntax
Switch (expression)
Case value1:
// Code to execute when the expression equals value1
Case value2:
// Code to execute when the expression equals value2
...
End Switch;
11/24
Example: Using the Switch Statement
Let’s take a look at an example of how the switch statement can be used to make

Introduction to Flowcharts
A flowchart is a visual representation of an algorithm or a process. It uses various
shapes and symbols to represent different elements and actions in a structured
manner. Flowcharts help in visualizing and understanding the logical flow of a
program.
Key components of a flowchart include:
• Start/End Symbol: Indicates the start or end of a process.
• Process Symbol: Represents an action or operation.
• Decision Symbol: Depicts a decision point with alternative paths.
• Input/Output Symbol: Represents input or output operations.
• Connector Symbol: Connects different parts of the flowchart.

Loops
Introduction to Loops in Programming
Loops are a fundamental concept in programming, and they play a crucial role in
automating repetitive tasks. Consider the following scenarios:
• Printing numbers from 1 to 10.
• Processing each item in a list.
• Repeating an action until a specific condition is met.
Loops allow us to efficiently address these situations, making our code more
powerful and flexible.
17/24
Syntax of Loops
Loops in programming consist of three main parts:
1. Initialization: Setting up initial values or conditions.
2. Condition: Defining the condition that must be true for the loop to continue.
3. Iteration: Specifying how the loop variable changes with each iteration.
Example of a basic loop structure:
<Start loop with termination condition>
// Loop body: Code to be executed repeatedly
<End loop>
<Start loop>
// Loop body: Code to be executed repeatedly
<End loop with termination condition>
18/24
Types of Loops
In programming, there are three common types of loops:
• For loop: Used for a predetermined number of iterations.
• While loop: Continues as long as a specified condition is true.
• Do-while loop: Similar to a while loop but guaranteed to execute at least once.
Each type of loop has its own use cases and advantages, depending on the situation
you are addressing in your code.

Special case of Arrays : String
In algorithms, a string is an array of characters. It’s a data structure that holds a
sequence of individual characters, such as letters, digits, or symbols.
The string "Hello" can be represented as an array: [’H’, ’e’, ’l’, ’l’, ’o’].
• The symbol " is used to delimit a string of characters.
• The string "Baobab" is composed of the characters ’B’, ’a’, ’o’, ’b’, ’a’, and ’b’.
• The string "123" consists only of digits, but it should never be confused with the
numerical value 123.
• The string "" represents an empty string.
• The string " " is a string containing only one character, which is the space character
(not to be confused with an empty string).

the 7-bit version and 256 characters in the 8-bit version (extended ASCII).

Comparing String
In algorithms, strings are often compared character by character using their ASCII
codes.
• Each character in a string is represented by a numeric ASCII code, which allows for
lexicographic comparison.
• Smaller ASCII code values indicate earlier positions in the ordering.
Example
• Compare "apple" and "banana."
• Compare characters one by one:
’a’ (97) < ’b’ (98), hence, "apple" is lexicographically smaller than "banana."
• This is how many algorithms sort and compare strings.

Selection Sort
Selection Sort Algorithm
Principle
Selection Sort is a simple sorting algorithm.
• It divides an array into two parts: sorted and unsorted.
• It repeatedly selects the minimum element from the unsorted part and moves it
to the sorted part.
• This process continues until the entire array is sorted.
In other words,
• the minimum element in the array is searched for,
• and then swapped with the element that is currently located at the first position,
• Now the minimum element in the remaining unsorted array is searched for and put in
the second position,
• and so on.
4/16
Selection Sort Algorithm : Example, A = [7, 5, 4, 2]
5/16
Selection Sort Algorithm
Algorithm Selection_Sort;
Var i, j, min, p, T[5]: integer;
Begin \\ T must be filled first
For i <- 1 to 5 do
min <- T[i];
p <- i;
For j <- i+1 to 5 do
If(T[j] < min)
min <- T[j];
p <- j;
EndIf;
EndFor;
T[p] <- T[i];
T[i] <- min;
EndFor;
End;

Bubble Sort Algorithm
Bubble sort is based on the idea of repeatedly comparing pairs of adjacent elements and
then swapping their positions if they exist in the wrong order.
Principle
The bubble sort algorithm consists in progressively moving up the largest elements of
an array. Its principle is
• to invert two successive elements if they are not sorted in the right order,
• and to start again until no more elements can be swapped.
Assume that A is an unsorted array of elements. This array needs to be sorted in ascending
order.

Introduction to Binary Search
Definition
Binary Search (also called Dichotomy Search) is an efficient algorithm for find-
ing a target value within a sorted array.
Main Idea: Instead of checking every element one by one, we repeatedly divide
the search space in half, eliminating half of the remaining elements at each step.
1.1
Why Learn Binary Search?
• Much faster than linear search for large arrays
• Fundamental algorithm - appears everywhere in computer science
• Real-world applications: dictionaries, phone books, databases, file systems
• Foundation for more advanced algorithms
• Common in exams and technical interviews
1.2
The Key Requirement
Warning!
CRITICAL REQUIREMENT: Binary search ONLY works on SORTED ar-
rays!
If the array is not sorted, you must sort it first, or use linear search instead.

3
Binary Search Algorithm - Step by Step
3.1
The Algorithm in Words
Step-by-Step
To search for value X in sorted array T[1..n]:
1. Initialize:
• Set left = 1 (first index)
• Set right = n (last index)
2. Repeat while left ≤right:
• Calculate mid = ⌊(left + right) / 2⌋
• Compare T[mid] with X:
– If T[mid] = X →Found! Return mid
– If T[mid] > X →Search left half: set right = mid - 1
– If T[mid] < X →Search right half: set left = mid + 1
3. If loop ends without finding X:
• X is not in the array
• "not found"

Important Concepts and Details
8.1
Why Must the Array Be Sorted?
Example
Example with UNSORTED array: [15, 3, 9, 21, 7, 12, 18] Searching for 7:
• mid = 4, T[4] = 21
• 21 > 7, so we search left half: [15, 3, 9]
• But 7 is actually in the RIGHT half!
• Binary search FAILS on unsorted data
Example with SORTED array: [3, 7, 9, 12, 15, 18, 21] Now searching for 7
works correctly because the ordering guarantees:
• If T[mid] > 7, then ALL elements after mid are > 7
• If T[mid] < 7, then ALL elements before mid are < 7

2 right <- mid;

Summary and Key Points
Binary Search Summary
Algorithm Type: Divide and Conquer, Iterative Main Steps:
1. Start with left = 1, right = n
2. Calculate mid = (left + right) div 2
3. Compare T[mid] with target X:
• If equal →Found!
• If T[mid] » > X →Search left: right = mid - 1
• If T[mid] « < X →Search right: left = mid + 1
4. Repeat until found or left » > right
Key Requirements:
• Array MUST be sorted
• Use integer division for mid
• Loop condition: left ≤right
• Update: right = mid - 1 or left = mid + 1
12.1
Memory Aid: The 3 Rules
1. SORTED: Array must be sorted (ascending or descending)
2. MIDDLE: Always check the middle element first
3. HALVE: Cut the search space in half each time

Part 1: Algorithm Fundamentals
1.1
What is an Algorithm?
An algorithm is:
• A finite sequence of well-defined, unambiguous, executable steps
• A systematic approach to solving a problem
• A set of instructions that transforms input into output
Key Properties:
1. Must have a finite number of steps
2. Must terminate after finite operations
3. Must provide a result
4. Steps follow in a certain order
5. Each step must be unambiguous
1.2
Algorithm vs. Program
• Algorithm: Human-readable description of steps
• Program: Algorithm translated into a programming language
TIP
In this course, we write algorithms in French-style pseudocode using English
words. This is NOT a programming language.

Three essential parts:
1. Header: Algorithm name
2. Declarations: Variables and Constants
3. Body: Instructions between Begin and End
WARNING
EVERY algorithm MUST declare variables before any instructions.
